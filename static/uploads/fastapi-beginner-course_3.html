1 What is an API? (And why FastAPI?)

What is an API?

API stands for Application Programming Interface. Think of it as a waiter in a restaurant:

  • You (the customer) = a website or mobile app
  • The kitchen = the database/server where data lives
  • The waiter (the API) = carries your request to the kitchen and brings food back

You never walk into the kitchen yourself. You ask the waiter, and the waiter handles it. An API works exactly the same way — your app asks the API for data, and the API responds.

What is FastAPI?

FastAPI is a Python framework (a toolkit) for building APIs quickly. It is:

  • Fast — one of the fastest Python frameworks available
  • Easy — very little code needed
  • Automatic — it creates interactive documentation for you for free
  • Modern — uses modern Python features that catch bugs early

We will see all of these benefits as we go.

2 Setting Up Your Environment

Before writing code, we prepare our computer. Do these steps in your terminal (Command Prompt, PowerShell, or Mac/Linux Terminal).

Step 1 — Check Python is installed

python --version

This should show something like Python 3.11.x. FastAPI needs Python 3.8 or higher. If you don't have Python, download it from python.org.

Step 2 — Create a project folder

mkdir my_first_api
cd my_first_api
  • mkdir = "make directory" (creates a folder named my_first_api)
  • cd = "change directory" (move into that folder)

Step 3 — Create a virtual environment

A virtual environment is a private, isolated box for your project's tools. It keeps this project's libraries separate from other projects so they don't conflict.

python -m venv venv
  • python -m venv = run Python's built-in venv tool
  • the second venv = the name of the folder it creates

Activate it:

# On Windows:
venv\Scripts\activate

# On Mac/Linux:
source venv/bin/activate

After activating, you'll see (venv) at the start of your terminal line. That means the box is open and active.

Step 4 — Install FastAPI

pip install "fastapi[standard]"
  • pip = Python's package installer (it downloads libraries for you)
  • fastapi[standard] = installs FastAPI plus the extra tools it commonly needs (including the server that runs your app)

You're ready to write code.

3 Your First FastAPI App — Line by Line

Create a file called main.py and type this:

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello, World!"}

That's a complete, working API. Let's break down every single line.

Line 1: from fastapi import FastAPI

This is an import statement. In Python, import brings in code that other people wrote so you can use it.

  • fastapi = the library we installed
  • FastAPI = a specific tool inside that library (it's the main class that builds our app)

Read it as: "From the fastapi library, bring in the FastAPI tool."

🐍 Python Concept — what is a "class"?A class is a blueprint for creating objects. FastAPI is a blueprint for creating an API application. When we use the blueprint, we get an actual app we can work with.

Line 3: app = FastAPI()

Here we create our application from the blueprint.

  • FastAPI() = the parentheses () mean "build one now" (this is called instantiating)
  • app = = we store the result in a variable named app
🐍 Python Concept — variablesA variable is a labeled box that holds a value. app = FastAPI() means "put the new FastAPI application into a box labeled app." From now on, whenever we write app, we mean that application.

Line 5: @app.get("/")

This line is a decorator (notice the @ symbol). It tells FastAPI: "When someone visits the URL path / using a GET request, run the function written directly below me."

  • @app.get = "listen for a GET request" (GET = "give me data")
  • ("/") = the path. / is the root — the homepage of your API

Lines 6–7: The function

def read_root():
    return {"message": "Hello, World!"}
  • def = "define a function"
  • read_root = the name we chose
  • () = inputs the function takes (none here)
  • : = marks the start of the function's body
🐍 Python Concept — dictionariesA dictionary stores data as key: value pairs inside curly braces {}. Here the key is "message" and the value is "Hello, World!". Ask for the label "message", get back "Hello, World!".

FastAPI automatically converts this dictionary into JSON — the standard text format APIs use to send data.

Running your app

fastapi dev main.py

You'll see a URL, usually http://127.0.0.1:8000. Open it in your browser.

💡 What is 127.0.0.1:8000?127.0.0.1 (also called localhost) means "this computer." :8000 is the port — like a door number where the app is listening. So you're visiting your own machine, door 8000.

You should see {"message":"Hello, World!"}. You just built an API.

Bonus: Free automatic documentation

Visit http://127.0.0.1:8000/docs. FastAPI built an interactive testing page for you — automatically, with zero extra code.

★ A Simple Example API (Full Working File)

Before we go deeper, here is one small but complete API you can copy, run, and play with right now. It has three endpoints and uses everything from Part 3. Don't worry if a line looks new — we explain each idea in the parts that follow.

Create a file called main.py and paste this:

from fastapi import FastAPI

# 1. Create the application
app = FastAPI()

# 2. A homepage endpoint
@app.get("/")
def home():
    return {"message": "Welcome to my first API!"}

# 3. A greeting endpoint that uses a path parameter
@app.get("/hello/{name}")
def say_hello(name: str):
    return {"greeting": "Hello, " + name + "!"}

# 4. A simple calculator using query parameters
@app.get("/add")
def add_numbers(a: int, b: int):
    return {"a": a, "b": b, "result": a + b}

What each endpoint does

Visit this URLYou get back
/{"message": "Welcome to my first API!"}
/hello/Jobin{"greeting": "Hello, Jobin!"}
/add?a=5&b=3{"a": 5, "b": 3, "result": 8}

Run it

fastapi dev main.py

Then open http://127.0.0.1:8000 in your browser. Try changing the URL to /hello/Ralfiz or /add?a=10&b=20 and watch the response change instantly.

💡 Notice the magicIn /add, we wrote a: int and b: int. FastAPI automatically turns the text from the URL into real numbers and adds them. If you visit /add?a=hello&b=3, FastAPI rejects it with a clear error — you never wrote a single line of validation. That's the power you'll understand fully by Part 7.

This tiny file already shows the three ways data flows into an API: a plain path (/), a path parameter (/hello/{name}), and query parameters (/add?a=&b=). Keep this file handy — we'll build on these exact ideas next.

4 Understanding Functions & Decorators

Functions

A function is a named set of instructions you can run whenever you want.

def greet(name):
    return "Hello, " + name
  • def greet(name): — defines a function called greet taking one input called name
  • return "Hello, " + name — joins the text with whatever name is

If you call greet("Jobin"), you get "Hello, Jobin". The + joins strings together (called concatenation).

🐍 Python Concept — parameters vs argumentsThe name inside the parentheses when defining is a parameter (a placeholder). The actual value you pass in, like "Jobin", is an argument. Parameter = the empty slot; argument = what you put in it.

Decorators

A decorator is a function that wraps another function to give it extra powers, using the @ symbol on the line above.

In FastAPI, @app.get("/") doesn't change what your function returns — it registers your function with the app so it runs when a matching web request comes in. Just read it as:

"Hey FastAPI, connect this URL path to this function."

That mental model is enough to be productive.

5 Path Parameters

Often you want a URL to include a specific value — like a user ID. These are path parameters.

@app.get("/users/{user_id}")
def get_user(user_id):
    return {"user_id": user_id}
  • {user_id} in curly braces means "this part is a variable that changes"
  • The name in the braces must match the function parameter name

Try it: visit http://127.0.0.1:8000/users/42 and you'll get {"user_id": "42"}. Notice 42 arrived as text. To make it a real number, add a type hint:

@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}

By adding : int, FastAPI converts 42 to a real integer and rejects anything that isn't a number — automatically.

6 Query Parameters

Query parameters are extra options added after a ?. You've seen these: ?page=2&sort=price.

@app.get("/products")
def list_products(category: str = "all", limit: int = 10):
    return {"category": category, "limit": limit}
  • category: str = "all" — text, defaults to "all" if not provided
  • limit: int = 10 — number, defaults to 10

Because these aren't in the URL path, FastAPI treats them as query parameters automatically.

URLResult
/productscategory = "all", limit = 10 (defaults)
/products?category=shoescategory = "shoes", limit = 10
/products?category=shoes&limit=5category = "shoes", limit = 5
🐍 Python Concept — default valuesWhen you write limit: int = 10, the = 10 makes that parameter optional. If the caller doesn't provide it, Python uses 10. Without a default, the parameter would be required.

7 Python Type Hints (Important Foundation)

You've seen : int and : str. These are type hints, and they're a big reason FastAPI is so powerful.

What is a "type"?

TypeMeansExample
intInteger (whole number)42, -7
floatDecimal number3.14, 9.99
strString (text)"hello"
boolBoolean (true/false)True, False
listA list of items[1, 2, 3]
dictA dictionary{"name": "Jobin"}

What is a type hint?

age: int = 25
name: str = "Jobin"
price: float = 99.99
is_active: bool = True

The part after the colon is the hint. Plain Python doesn't force these rules, but FastAPI does — and that's the magic:

  • FastAPI reads your type hints
  • It automatically validates incoming data (rejects bad input)
  • It automatically converts data to the right type
  • It uses them to build that free documentation

So def get_user(user_id: int): guarantees user_id arrives as a real integer, or returns a clear error. Free input validation just by being descriptive.

8 Request Bodies with Pydantic

When creating something — say, a new user — we send a whole package of data. This is the request body, usually as JSON. FastAPI handles it using Pydantic.

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):
    name: str
    email: str
    age: int

@app.post("/users")
def create_user(user: User):
    return {"message": "User created", "data": user}

The User class

  • class User(BaseModel): — defines a data shape called User; (BaseModel) inherits validation powers
  • name: str — must have a name that is text
  • email: str — must have an email that is text
  • age: int — must have an age that is a whole number
🐍 Python Concept — inheritanceWhen a class is written as class User(BaseModel), User inherits from BaseModel — it automatically gets all of BaseModel's abilities (like validation) for free, the way a child inherits traits from a parent. We just add our specific fields on top.

The endpoint

  • @app.post("/users") — listens for a POST request (POST = "create new data")
  • def create_user(user: User): — expects an input matching the User shape; FastAPI reads the request body and validates it
  • If data is missing email, or age is text instead of a number, FastAPI automatically rejects it. You wrote zero validation code.

Testing it

Open http://127.0.0.1:8000/docs, find POST /users, click "Try it out", and send:

{
  "name": "Jobin",
  "email": "jobin@example.com",
  "age": 28
}

9 HTTP Methods — GET, POST, PUT, DELETE

When an app talks to an API, it uses an HTTP method to say what kind of action it wants. Think of them as verbs:

MethodMeaningAnalogy
GETRead / fetch dataReading a page in a book
POSTCreate new dataWriting a new entry
PUTUpdate existing dataErasing and rewriting an entry
DELETERemove dataTearing out a page
@app.get("/items")         # read items
@app.post("/items")        # create an item
@app.put("/items/{id}")    # update item with that id
@app.delete("/items/{id}") # delete item with that id

This pattern — using these four methods on resources — is the foundation of REST APIs, the standard style for building web APIs today.

10 Mini Project — A Complete To-Do API

Let's combine everything into one working application. It stores tasks in memory (a Python list) so we can focus on the API itself without a database yet.

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI()

# Our "database" — just a Python list for now
todos = []

# The shape of one to-do item coming IN (from the user)
class TodoCreate(BaseModel):
    title: str
    completed: bool = False

# The shape of a to-do item going OUT (includes an id)
class Todo(BaseModel):
    id: int
    title: str
    completed: bool

# CREATE a new to-do
@app.post("/todos")
def create_todo(todo: TodoCreate):
    new_todo = {
        "id": len(todos) + 1,
        "title": todo.title,
        "completed": todo.completed,
    }
    todos.append(new_todo)
    return new_todo

# READ all to-dos
@app.get("/todos")
def get_todos():
    return todos

# READ one to-do by id
@app.get("/todos/{todo_id}")
def get_todo(todo_id: int):
    for todo in todos:
        if todo["id"] == todo_id:
            return todo
    raise HTTPException(status_code=404, detail="Todo not found")

# UPDATE a to-do
@app.put("/todos/{todo_id}")
def update_todo(todo_id: int, updated: TodoCreate):
    for todo in todos:
        if todo["id"] == todo_id:
            todo["title"] = updated.title
            todo["completed"] = updated.completed
            return todo
    raise HTTPException(status_code=404, detail="Todo not found")

# DELETE a to-do
@app.delete("/todos/{todo_id}")
def delete_todo(todo_id: int):
    for index, todo in enumerate(todos):
        if todo["id"] == todo_id:
            todos.pop(index)
            return {"message": "Deleted successfully"}
    raise HTTPException(status_code=404, detail="Todo not found")

Walking through the new pieces

todos = [] — an empty list. A list [] holds many items in order. This is our temporary database.

Two classes — the user shouldn't send an id (the server assigns it), so the incoming shape (TodoCreate) differs from the complete shape (Todo).

Inside create_todo:

  • len(todos) — len() counts items in the list; we add 1 for a simple new id
  • todo.title — the dot . means "reach inside this object and grab this piece"
  • todos.append(new_todo) — .append() adds an item to the end of a list
🐍 Python Concept — for loopsA for loop repeats an action for every item in a collection. for todo in todos: means "go through the todos list one item at a time; call the current one todo."
🐍 Python Concept — if and ==if runs code only when a condition is true. == checks if two things are equal (two equals signs for comparison; one = is for assigning values — don't mix them up).

raise HTTPException(...) — if the loop finishes without a match, we trigger an error on purpose. raise = trigger an error, 404 = standard "Not Found" code, detail = a friendly message.

🐍 Python Concept — enumerateNormally a for loop gives just the item. enumerate() gives you both the position number (index) and the item, so we can remove the right one with todos.pop(index).
✅ CRUDCreate, Read, Update, Delete — the four basic operations almost every app needs. You just built all four.

★ How to Test Your API

Writing an API is only half the job — you need to test it to confirm each endpoint works. There are four common ways, from easiest to most powerful. Beginners should start with the first two.

1. The browser (GET requests only)

The simplest test. Just type the URL into your browser's address bar:

http://127.0.0.1:8000/add?a=5&b=3

The browser shows the JSON response directly. This works for GET requests only — you can't easily send POST, PUT, or DELETE this way, because the browser bar always sends GET.

2. The automatic docs — /docs (recommended for beginners)

This is FastAPI's superpower. Open this URL in your browser:

http://127.0.0.1:8000/docs

You get a full interactive testing page (called Swagger UI) built automatically. To test any endpoint:

  1. Click the endpoint you want (e.g. POST /todos)
  2. Click the "Try it out" button
  3. Fill in any parameters or JSON body
  4. Click "Execute"
  5. Scroll down to see the real response, status code, and headers

This handles every HTTP method (GET, POST, PUT, DELETE) with no extra tools. For a beginner, this is the best place to test everything.

💡 Bonus docs pageFastAPI also gives you a second style of docs at http://127.0.0.1:8000/redoc — cleaner for reading, though you can't click-to-test there. Use /docs for testing, /redoc for browsing.

3. The command line — curl

curl is a tool built into most computers for sending requests from the terminal. Useful for quick checks and scripts.

A GET request:

curl http://127.0.0.1:8000/todos

A POST request (creating a to-do) — we send JSON data:

curl -X POST http://127.0.0.1:8000/todos \
  -H "Content-Type: application/json" \
  -d '{"title": "Learn FastAPI", "completed": false}'
  • -X POST — choose the HTTP method (default is GET)
  • -H "Content-Type: application/json" — a header telling the server we're sending JSON
  • -d '{...}' — the data (the request body) we're sending

A DELETE request:

curl -X DELETE http://127.0.0.1:8000/todos/1

4. A GUI tool — Postman or Thunder Client

For real-world projects, developers use a visual tool to organize and re-run requests:

  • Postman — a popular standalone app. Create a request, pick the method from a dropdown, add headers and a JSON body in nice form fields, then hit Send.
  • Thunder Client — a lightweight extension that lives inside VS Code, so you never leave your editor.

These let you save collections of requests, reuse them, and share them with your team — ideal once a project grows.

5. Automated tests in code (a peek ahead)

Eventually you'll want tests that run automatically so you catch bugs before users do. FastAPI makes this easy with a built-in TestClient:

from fastapi.testclient import TestClient
from main import app   # import the app from your main.py

client = TestClient(app)

def test_home():
    response = client.get("/")
    assert response.status_code == 200
    assert response.json() == {"message": "Welcome to my first API!"}
  • TestClient(app) — creates a fake client that calls your API in memory (no server needed)
  • client.get("/") — sends a GET request, just like a browser would
  • assert — a Python keyword meaning "this MUST be true, or fail the test"
  • response.status_code == 200 — check we got a success code
  • response.json() — read the JSON the API sent back and compare it
🐍 Python Concept — assertassert checks that a condition is true. If it is, nothing happens and the test passes. If it's false, Python raises an error and the test fails — telling you something broke. It's how automated tests "check" your code.

Which one should you use?

SituationBest tool
Just learning / quick GET checkBrowser
Testing any endpoint while building/docs (start here)
Quick terminal check or scriptingcurl
Organized, repeatable testing on a real projectPostman / Thunder Client
Catching bugs automatically before releaseTestClient (code)

As a beginner, live in the /docs page. It's free, requires nothing to install, and tests everything.

11 Handling Errors Properly

Good APIs return clear, correct errors using status codes — numbers that tell the client what happened:

CodeMeaningWhen
200OKEverything worked (default for GET)
201CreatedSomething was successfully created
400Bad RequestThe client sent invalid data
401UnauthorizedNot logged in
403ForbiddenLogged in but not allowed
404Not FoundThe thing doesn't exist
500Server ErrorSomething broke on our side

Raising a clean error

from fastapi import HTTPException

if not user_found:
    raise HTTPException(
        status_code=404,
        detail="User not found"
    )

This sends a proper 404 with a clear message instead of crashing. FastAPI also automatically returns 422 errors when incoming data fails type validation — again, free.

Setting a custom success code

@app.post("/todos", status_code=201)
def create_todo(todo: TodoCreate):
    ...

Just add status_code=201 to the decorator.

12 What to Learn Next

You now understand the core of FastAPI. Here's a roadmap from beginner to production-ready:

  1. Databases — Replace the in-memory list with a real database. Learn SQLAlchemy and connect it to PostgreSQL.
  2. Async / await — FastAPI supports asynchronous code (async def) for handling many requests efficiently.
  3. Authentication — Protect endpoints with JWT tokens so only logged-in users can access certain routes.
  4. Project structure — Split one big main.py into organized folders using routers (APIRouter).
  5. Environment variables — Keep secrets out of code using a .env file and Pydantic settings.
  6. Deployment — Put your API online so the world can use it.
